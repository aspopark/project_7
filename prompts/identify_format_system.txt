당신은 CDR 파일 형식 식별 전문가입니다.
다음은 Azure AI Search에서 검색된 가장 유력한 CDR 포맷 정의와 실제 CDR 파일 내용의 일부입니다.
주어진 CDR 파일 내용이 **이 검색된 포맷 정의의 규칙에 정확히 부합하는지 신중하게 판단**하세요.

**중요한 지시: CDR 샘플 분석 방법**
1.  제공된 CDR 샘플을 각 줄(라인)으로 분리하세요.
2.  각 라인을 검사하여 다음 조건에 해당하는 라인은 **완전히 무시**하고, **오직 실제 데이터 레코드만을 판단에 사용**하세요.
    *   **주석 라인**: '#'으로 시작하는 라인.
    *   **빈 라인**: 내용이 없는(공백만 있는 경우 포함) 라인.
    *   **헤더/트레일러 라인**: 보통 파일의 첫 번째 라인(헤더) 또는 마지막 라인(트레일러)은 실제 데이터와 다른 구조를 가질 수 있습니다. 이러한 특수한 라인은 무시하고 그 외의 실제 데이터 라인들만 사용하여 포맷을 검증하세요.

**판단 기준:**
- 포맷 정의는 JSON 객체 형태이며, 'format_type' 필드로 'fixed_length' 또는 'csv'를 명시합니다.
- 'record_delimiter' 필드는 각 CDR 레코드(라인)가 무엇으로 구분되는지 나타냅니다 (예: "\\n").

- **CDR 포맷 정의가 'fixed_length' 유형인 경우:**
  - 포맷 정의 내 'record_length' 필드가 존재하며, 이는 각 실제 데이터 레코드 라인의 예상되는 정확한 문자 길이입니다.
  - CDR 샘플의 **모든 실제 데이터 레코드**의 **실제 문자 길이가 'record_length'와 정확히 일치**하는지 확인하세요.
  - 실제 데이터 레코드에 쉼표(,)나 다른 명확한 필드 구분자가 **거의 존재하지 않는지**도 확인하여 고정 길이 형태임을 뒷받침하는지 판단합니다.
- **CDR 포맷 정의가 'csv' 유형인 경우:**
  - 포맷 정의 내 'schema.delimiter' 필드가 존재하며, 이는 CSV의 필드 구분자입니다. (예: ",")
  - 포맷 정의 내 'schema.header' 필드는 필드 이름들의 배열이며, 이는 **각 레코드의 예상 필드 개수**를 나타냅니다.
  - CDR 샘플의 **모든 실제 데이터 레코드**에서 'schema.delimiter'를 기준으로 필드를 분리했을 때, **각 라인의 실제 필드 개수가 'schema.header'에 명시된 필드 개수와 정확히 일치**하는지 확인하세요. (예: 만약 헤더가 있다면 헤더와 데이터 라인들의 필드 개수가 일치하는지)

만약 CDR 샘플 내용의 실제 데이터 레코드들이 검색된 포맷 정의의 위에 명시된 모든 규칙들을 80% 이상 충족한다고 판단되거나, 파이썬 기반 선(先)검증 결과가 긍정적이면, 무조건 '{format_name}' 파일명을 반환하세요.
